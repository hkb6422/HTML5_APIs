Geolocation

The geolocation API defines a high level interface that allows us to retrieve location information about a hosting device. Common sources of location information include the Global Positioning System (GPS) and information inferred from network details such as Wi-Fi or a devices IP address.

Throughout this tutorial you will learn how to obtain location information from your device, plot the location on Google Maps and also how to update location information as a device is moving.

Testing for Geolocation support

Before trying to retrieve the location of your device we firstly need to test whether the browser being used actually supports the geolocation API.

So how do we access the geolocation API to test if it is supported? The geolocation API is published through the navigator.geolocation object. The navigator object contains information about the browser being used and this includes the geolocation property. Invoking the geolocation property will return a reference to the geolocation object. A reference will only be returned if the geolocation API is supported!

Geolocation Methods

Previous Exercise: We have created an if and else statement to test for browser support. Users are alerted with a message if the geolocation API is not supported. Now we will explore the methods exposed by the geolocation object which can be accessed if the API is supported.


The image above shows all of the methods that the geolocation object contains. Let's explore each of these methods and identify the differences between them and what they do.

getCurrentPosition

The getCurrentPosition is used to get the current position of a device and the request will only be executed once. If location retrieval is successful, a Position object is returned with all the relevant information.

watchPositon

The watchPosition method is used to register a handler function that will continuously be executed each time a device's location changes. The method will return a Position object and a unique transaction ID each time the location updates.

clearWatch

The clearWatch method is used to unregister the handlers previously declared using the watchPostion method.

Position object

Previous Exercise: We have learnt about the getPosition and watchPosition methods and the parameters used. Now we will explore the position object that is returned by the methods and what properties it contains.

The position object is returned from the watchPosition and getPosition methods if locating the device is successful. The object contains location information regarding the device we are using. The position object contains a timestamp and coords property. Take a look at the table below to see what information we can retrieve using these two properties.


Coords property

The position.coords property will return a coordinates object which defines the current location. We are able to access any of the properties by specifying the position.coords. followed by the property name. The three properties which are always guaranteed to be returned are the latitude, longitude and accuracy. Other coords properties are dependent on the device you are using and will return null if they are not supported

Timestamp property

The timestamp property will return a DOMTimeStamp which will represent the time in which the location information was retrieved.

Error Object

Previous Exercise: We have retrieved and displayed location information about our device. However sometimes an error may occur when retrieving these details. We will now explore how to identify and display errors if they occur

A PositionError object is returned if the retrieval of location is unsuccessful and contains two properties named message and code. The table below shows all of the error details which the PositionError object contains.

Code Property

The code property will return a integer value which will indicate what type of error occurred when trying to retrieve the location details. The property is accessed using the statement error.code

Message Property

The message property will return a string which describes the details of the error. The property is accessed using the statement error.message

Google Maps

Previous Exercise: You have retrieved your location with customised options. In this exercise you will learn how to plot the location information retrieved onto a map using the Google Maps API.

The Google Maps JavaScript API allows users to embed Google Maps into their web pages. The API allows users to customize the look of the map and choose what information is shown.

In order to access the Google Maps API you will need to include the source of the API shown in the image below into your code.

Map

The Maps object is stored within an object called google and this is used for all Google objects. To access any object relating to Google Maps we always start with google.maps

To add a map to the page we need to create a new Map object by calling google.maps.Map. The map class accepts two parameters that are;


A HTML element that you wish to draw the map on - this is usually a div element with an assigned id


A google options object which controls how the map is displayed using various properties.
