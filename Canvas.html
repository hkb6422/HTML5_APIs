Canvas

The canvas is a HTML element which can be used to draw graphics by using scripting in JavaScript. The canvas for instance can be used to draw graphics, create simple animations and alter images via pixel manipulation. Throughout this tutorial you will learn about the 2D context the canvas offers and different drawing methods that can be applied to the canvas.

Canvas Basics

Canvas Element

The canvas element at first sight looks like an img element with the noticeable difference of not having an alt attribute. The canvas however does have two attributes this being height and width which will set the dimensions of the canvas. These attributes are optional however when no width and height is specified a default setting of 300 pixels wide by 500 pixels high is applied. A canvas is also usually given an id attribute so it is easily accessible like any other element.

The canvas by default has no border or content, but can be styled like any other image by using margins and borders etc. A border is usually applied to make the canvas visible by applying a thin frame. However remember this will not style the actual drawing on the canvas but the canvas itself.

Canvas Grid

In a 2D space positions are referenced using X and Y coordinates. The Y axis extends vertically and the X axis extends horizontally. In the canvas coordinate system the origin of (0,0) is placed in the upper-left corner opposed to the centre. The X coordinates will increase to the right and the Y coordinates increase towards the bottom. Hover over the canvas to the left and see what coordinates are given.

Checking for support

Just like the Geolocation API, we can also test whether the browser supports the canvas API. To test for the API we simply test for the presence of the getContext() method. If the browser supports the canvas API drawing code can be executed, if the browser does not support the API users should be supplied with an alert.

Rendering Context

The canvas element creates a drawing surface that exposes a rendering context, which is used to create and manipulate the context displayed. Within this tutorial we will focus on the 2D rendering context. This context is where all drawing methods and properties are defined. These properties are explored in the exercises to follow.

Initially the canvas is blank and we need to access the rendering context to be able to display something. A method called getContext() is used to obtain the rendering context and all of the drawing methods available. The method accepts one parameter which will set the type of context, in this tutorial we will be using 2D which will return a CanvasRenderingContext2D.

Curves

Bezier Curve

A Bezier curve is a curve which has a start (P0) and end point defined (P3) which dictates the length of the curve. The curve has a unique feature of having two control points — one for each end of the line (P1 and P2). Bezier curves allow users to create more complex curvatures due to the increased flexibility of manipulating two control points of the curve.

In order to create a Bezier curve on the canvas the BezierCurveTo method is applied to the canvas context. The method requires six parameters ctx.BezierCurveTo(cp1x,cp1y,cp2x,cp2y,x,y). The first two parameters are the x and y position of the first control point, the next two parameters define the x and y position for the second control point. Finally the fifth and sixth parameters defines the start point of the curve on the x axis and the end point on the y axis.

Use the interactive example on the left to manipulate the Bezier curves two control points.

Quadratic Curve

A quadratic curve again has a start (P0) and end point (P2) like the Bezier curve that defines the length of the curve. There is one distinct difference between the two curves which is the control points. The Quadratic curve has only one control point not two (P1).

In order to create a quadratic curve the quadraticCurveTo method can be applied to the canvas context. The method takes four parameters ctx.quadraticCurveTo(cpx,cpy,x,y) - the first two are the x and y coordinates of the first control point and the last two parameters are a point on the x axis as the starting point of the curve and a point on the y axis which defines the ending point of the curve.

Use the interactive example to the left to see how the control point manipulates the quadratic curve.

Lines

To draw a line we need to use four methods which are defined in the image below. Take a look at the code to see how we draw a simple line onto the canvas.

Line Width

To define the width of a line the lineWidth property of the canvas context is used. A value defines the width of the line in pixels. The line width needs to be set before the stroke method is called

Line Colour

To set the colour of a line the strokeStyle property of the canvas context is used. The property accepts a colour as a string, hex value or RGB value

Line Cap

The lineCap property sets the end caps of a line. There are three available properties which are butt, round and square. The default value is set to butt if the property is not specified.

Arcs

Radians

Before delving into creating arcs, we firstly need to understand what radians are in order to draw the arcs. Radians is an angular measure just like degrees that describes the ratio between the length of an arc of any circle and its radius. Unlike degrees radians is a pure measure based on the radius of the circle. The size of a radian is determined by the requirement that there are 2π radians in a circle. Therefore 2π radians equals 360 degrees and π radians equals 180 degrees. This means that 1 radian = 180/π degrees, and 1 degree = π/180 radians

If you are more comfortable with using degrees - use the conversion calculator to convert degrees into radians. The values have been rounded to 2 decimal places for convenience


The image above shows the most used angles within a circle and each of the angles in degrees and radians

Arc


Creating an arc is like creating a section of the circumference of an imaginary circle. The imaginary circle is defined by a radius which determines the circumference of the circle and a centre point which is defined by the x and y coordinates of the canvas.

Once this imaginary circle is defined, we can start to build the arc itself. The arc is defined by the startAngle and endAngle. These two angles form imaginary lines that will start from the centre of the circle and create two end points of the arc in we wish to create. Remember these angles are in radians.

The last argument defines in which direction the arc is drawn. If it is set to false the arc will be drawn clockwise and true will cause the arc to be drawn anti-clockwise

Use the interactive example above and change the arc method parameters to draw an arc. The example will only accept numerical values.

Circles

The arc method can also be used to draw circles. A full circle responds to 2π which is approximately 6.28 radians therefore half a circle will respond to π which is approximately 3.14 radians.

To draw a circle you will need to;

1) Change the end angle to 6.28 - this will define the circumference of the circle
2) Define the start angle to 0 - this will ensure that the circle will begin from the centre
3) Change the direction to clockwise - this is done by setting the last parameter to false
4) Add the x and y positions to determine the position the circle is drawn

Use the interactive arc method example and change the parameters to experiment drawing circles. (You will need to use radians)

Pixel Manipulation

The canvas API allows users to manipulate individual pixels of an image. You are able to retrieve each pixel from the image access the RGBA channels: red, green, blue and alpha (the transparency channel).

Draw Image



The first step in drawing an image on the canvas is to create a canvas and access the 2D context. Remember to test whether the browser supports the canvas before invoking any methods.

To draw the image onto the canvas context we use the drawImage method which requires an image object and an x and y position in which the image is placed e.g. drawImage(image, x, y). The drawImage method requires an image object, the image needs to be loaded before calling the drawImage method which is accomplished using the onload event handler. To create an image object simply call the new Image() method and pass it the source path of the image.

Get Image Data





In order to manipulate the pixels of the image, we first need to retrieve all of the images data. The imageData method is used to retrieve the underlying pixel information of the area of the canvas. This is denoted by the rectangle which starts at the point (sx, sy) and has a height and width (sw, sh) ctx.getImageData(sx, sy, sw, sh)

Next, after the image object is returned from the above method, we want to access the data property. The data property contains a one dimensional array containing the pixel data in RGBA order with an assigned integer value from 0 to 255. RGBA is the four colour channels the image is made of - red, blue, green and alpha as shown in the image to the left. Each four byte value is representative of each pixel within the image.

To access the RGBA values we first need to create a loop which iterates over each four byte value within the array. As the RGBA values are in order, the first index will always be red, second is green, third is blue and fourth is alpha. Using these indexes we can access and manipulate certain details of each pixel as necessary. Take a look at the loop to the left which accesses each different pixel channel.

The final step after all manipulations are completed to the array, is to draw the image back onto the canvas. The putImageData method will copy the pixel data retrieved from the getDataImage method (which is stored in the imageInfo variable in the example) and draw this image back onto the canvas. The required parameters for the putImage method is the image array and the x and y position to draw the image putImage(image, x, y)

Gradient

Linear Gradient

The createLinearGradient method accepts four arguments. The first two parameters specify the x and y position of the starting point of the gradient whilst the last two parameters specify the x and y position of the ending point of the gradient. ctx.createLinearGradient(x0,y0,x1,y1)

To add a colour stop which is the position of any colour within the shape the addColorStop(stop,colour) method is used which takes two parameters. The first parameter is a value between 0 and 1 which represents the position between the start and end in a gradient. You can call the addColorStop method multiple times to alter the gradient. The second parameter is the colour that will be applied to the stop and can be written as an rgb value, hex value or as a string.

Once the gradient is completed it is simply applied using the fillStyle method which applies the assigned style to the object specified. Use the interactive example to apply a linear gradient with two colour stops by sliding the RGB values.

Radial Gradient

A radial gradient is applied using the createRadialGradient method and accepts six arguments ctx.createRadialGradient(x0,y0,r0,x1,y1,r1).The first three parameters define the first circle with coordinates (x0,y0) and radius(r0). The next three parameters define another circle with coordinates(x1,y1)and radius(r1). These two circles defined will create a circular gradient
In the example demonstrated above the createRadialGradient method has six parameters defined to create a radial gradient. The first parameters define a circle with a radius of 5 and the position is 75 pixels from the left and 50 pixels down from the top of the canvas. The second circle is defined by the next three parameters - so now we create a circle with radius of 100 that is 90 pixels from the left of the canvas and 60 pixels down from the top

All of the methods that can be applied to the linear gradient are also applicable to the radial gradient. You are again able to add colour stops by using the addColorStop() method. Use the interactive example and experiment applying a radial gradient with two colour stops by sliding the RGB values.
