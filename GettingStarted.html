Getting Started

The following section aims to provide you with all of the information needed for you to begin learning about the HTML5 APIs offered on this website. Throughout this section you will understand how the site is structured, the various features that are offered and what pre-existing knowledge is needed.


  • Prerequisites
  • Structure
  • Browser Editor

Prerequisites

In order learn about the HTML5 APIs and complete the exercises available throughout the tutorial there are a number of prerequisites shown below. Basic knowledge of the languages listed below is expected;

Language Description

HTML5

 HTML5 is the latest revision of the HTML standard which is used to structure content on the web. As the API's introduced in this website are part of the new HTML5 standard, knowledge of HTML5 is necessary in order to access and use the APIs.

CSS

 CSS is a language used to apply style to elements within the HTML mark-up. The language is not explored in the exercises as the APIs are not reliant on this language. However it is recommended to utilise CSS to make your creations look more personalised

JavaScript

 JavaScript is a language commonly used to program the behaviour of web pages. The HTML5 API interfaces are exposed through HTML5 and JavaScript therefore JavaScript is required in order to access the functionalities that they offer

Structure

Each page in the website is dedicated to a particular API. This page is split further into different topics which are accessible via the module tabs at the top of the page. The tabs are presented in sequential order therefore you view each page consecutively. The structure of each topic page is described below;


Quiz

Each page includes a quiz which is accessible by clicking the Quiz button. The number of questions vary from topic to topic. For each question there are a variety of answers which are each represented by a radio button. Each question will therefore have only one answer. The quiz essentially acts as a mechanism for you to test your knowledge before moving on and applying the knowledge learnt in completing any exercises.


Exercise

Each topic will also include an exercise for you to complete which will incorporate all of the knowledge learnt from the topics read so far. The exercises allow you to gain a first-hand experience of accessing and using the APIs. If you are looking for some help or guidance on how to complete the exercise you are able to retrieve some hints by clicking the See Hint button. The Edit Code button provides an in browser editor where you are able to complete the exercise given. This is explained in the next section.


Progress Bar

A progress bar is available to help you keep track of which exercises you have completed. Each different exercise has been assigned a score of either 10%, 15% or 20% depending on its complexity. Simply press the complete button to signify you have completed the exercise and the progress bar will update and present the percentage completed. Click the undo button to cancel any progress submitted for an exercise and the progress bar will reflect this.


Browser Editor

Within the website is a browser editor which allows you to write and run your code within the website. The browser editor has three panels which are dedicated to a separate language. There are three languages which are available to code in - HTML, JavaScript and CSS. The panels are collapsible therefore if you want to close any of the panels just simply click on the title or header of the panel. The editors provides features such as syntax highlighting, auto-completion and also real time validation


Please note that when using the HTML editor all tags including the HTML tags can be included within your code. However within the CSS editor you should not enter the style tags and within the JavaScript editor the script tags should not be entered.



There are many controls which have been implemented within the browser editor window. Each of the controls are described below;

Controls

Run

The run button will execute the code currently typed within the editors and display the results within an iframe. Each time you click the run button the code within the editors is also saved to session storage.


Clear

The clear button will clear the editors of any code that is currently written within them and clear the iframe of any content that is already displayed. Also when the clear button is pressed all data saved within session storage is erased therefore you will not be able to retrieve any code that was previously written if this button is clicked.


Get Text

The Get Text control will open a modal window with your code added to the template below. Your JavaScript code will appear in between the script tags and CSS code will be in between the style tags. Please note that you must include both the head and body tags in your HTML code in order to display code in between these tags in the template.


Set Code

Each time you navigate to a different tab and reopen the browser editor all code written prior to this is cleared from the editors. This control allows you to populate the editors with the code you had wrote in each of the editors before it was cleared. Please note that your code is only available within the current browser session.

See Answer

Finally within the browser editor window is a button called See Answer which presents the answer to the relevant exercise. The answers are provided in two formats a video and text. The video is a screen cast which demonstrates the answer to the exercise using the browser editor. The video includes annotations to help you understand the coding process. The code which is created in the screen cast to answer the exercise is presented next to the video.
