Drag and Drop

The drag and drop API is a powerful interface that makes it easier to copy, reorder and delete items using mouse clicks. This allows users to click and hold down a mouse button over an element and drag it to another location. When the mouse is released this will cause the element to be dropped within that location.

Before the drag and drop API the functionality of drag and drop was achieved using complex JavaScript programming or with other frameworks such as jQuery. The API now brings native support of drag and drop to the browser making it much easier to code

Drag and drop element

Create a draggable element and drop target

Before looking at the event handlers within the drag and drop API, we firstly need to create an element that can be dragged and an area that the element can be dragged into. This is done in two simple steps;

To make an element draggable the draggable="true" attribute is added to your content that is going to be dragged. In any DOM-compliant browser, a draggable element requires a "draggable" attribute which is set to "true" to enable this operation. By default images and link elements are draggable therefore no additional coding is needed for these elements.

To create a drop area this is simply done by creating a div element that will act as a drop area. The height and width of the div needs to be specified and it is also beneficial to add a border around the div to make it more visible.

Event Listeners

Now that we understand the order of which the event handlers are called in order for the browser to know what JavaScript to code to execute, we can begin to utilise these events.

An event listener can simply track whether any event handler the drag and drop API provides are called. If the event handler is called we are able to implement what happens if and when each of these event handlers are called. As demonstrated in the example we will be providing information about how many times the event is called during the drag and drop operation.

The first step is to implement the event listeners and the count of the events will follow in the next exercises

AddEventListener

AddEventListener is utilised to add event listeners to certain event handlers. The format of adding the event listener is provided below;

1) target -the target is element we want to attach an event handler on
2) type - The type refers to a string representing the event type to listen for
3) listener -The listener is the object that will receive a notification if the event occurs. This is usually a JavaScript function

Drag Event Listeners

To add the event listeners for the event handlers relating to the draggable image we need to firstly get a reference of the image. Next three event listeners are created one for each of the event handlers. The target of the event listeners is the image, the event which is being listened for are the API event handlers and finally the object that will receive the notification will be a JavaScript function created in the following exercises.

Drop Event Listeners

The same logic applied above is also carried out for the drop event handlers. The four event listeners are created for each of the event handlers associated with the drop operation. They all target the dropZone and the object receiving the notification will be a function which again will be created in the following exercises.

Drag functions

In the previous exercise we added event listeners for all of the event handlers of the drag and drop API. This section focuses on creating the functions for the events handlers relating to the draggable image.

DataTransfer object

Before creating any functions, we firstly need to understand how to set information relating to the drag and drop operation. In order to append the image onto the drop zone we need access to the image id.

The DataTransfer object is used to hold data about the elements that are being dragged during the drag and drop operation. This object is available from the dataTransfer property of all drag events. In order for the DataTransfer object to hold the id of the draggable image we need to use the setData method. The method will tell the DataTransfer object to hold the id of the image so we can access the image when the drop event handler occurs and then append the image using it's id onto the drop zone.

The setData method is called by stating the following;
object.setData(dataFormat,data);
1) object - Assigns data to the dataTransfer object accessed through the event object.
2) dataFormat - The dataformat is the format of the data - the id is text format
3) data - The data refers to the data to add. We want to add the id of the element that triggered the event - this being the image

handleStart function

The image above shows the handleStart function which controls what happens when a user starts dragging the image.

1) We firstly have to create a count variable which is incremented each time the dragStart event handler is called.
2) Next we add the setData method to store the image id
3) Finally we create a paragraph id and access it to display the count variable each time the dragStart handler is called

Drop functions

Before we begin creating the four functions to show the count of the dragleave, drop, dragenter and dragover events we firstly need to understand two addtional features that need to be added to the handledrop function

Browser Default


The preventDefault method needs to be the first method added to the handleDrop function which will listen for the drop handler. The method simply prevents the default action of the element from happening. In this case the default behaviour for an image is for it to be dragged, however we want the image to be appended to the drop zone.

Add image to drop zone


In order to be able to append the image to the drop zone we firstly need to access the image id from the DataTransfer Object that was set in the previous exercise. The method used to do this is demonstrated above - we simply call the getData method specifying the data format which is text. Next in order to actually append the image onto the drop zone we will need to use the.target.appendChild method. This method will append the image to the target which is the drop zone. We just need to simply supply the id of the image to this method.
